---
title: "Exams"
description: "List, view, start, submit, results, validate access, and stats"
---

All endpoints require `Authorization: Bearer <access_token>`.

### List exams

GET `/api/v1/exams`

### Get exam

GET `/api/v1/exams/:id`

### Validate access

GET `/api/v1/exams/:id/validate-access`

### Start exam session

POST `/api/v1/exams/:id/start`

### Submit answers

POST `/api/v1/exams/submit`

Request (shape simplified)
```json
{
  "session_id": "...",
  "answers": [
    { "question_id": "...", "response": "..." }
  ]
}
```

### Results

GET `/api/v1/exams/:id/results?session_id=...`

### Create exam (admin, instructor)

POST `/api/v1/exams`

### Stats (admin, instructor)

GET `/api/v1/exams/:id/stats`
