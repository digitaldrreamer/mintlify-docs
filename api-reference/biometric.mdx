---
title: "Biometric"
description: "Enrollment, verification, liveness, and session management"
---

All endpoints require `Authorization: Bearer <access_token>`.

### Register (multipart)

POST `/api/v1/biometric/register`

multipart/form-data
- image_data: file (image/*, <= 5MB)
- type: integer (0 face, 1 fingerprint, 2 voice)

### Start session

POST `/api/v1/biometric/session/start`

Request
```json
{ "continuous_monitoring": true }
```

### Active session

GET `/api/v1/biometric/session/active`

### End session

POST `/api/v1/biometric/session/end`

Request
```json
{ "session_id": "..." }
```

### Verify identity (multipart)

POST `/api/v1/biometric/verify`

multipart/form-data
- image_data: file
- type: integer
- session_id: optional (also `X-Session-ID` header or query)

### Continuous verify (base64 JSON)

POST `/api/v1/biometric/continuous-verify`

Request
```json
{ "session_id": "...", "image_data": "<base64>" }
```

### Liveness

POST `/api/v1/biometric/liveness`

Request
```json
{ "image_data": "<base64>", "challenge": null }
```

### Status

GET `/api/v1/biometric/status`

### Get record

GET `/api/v1/biometric/record/:recordId`

### Admin

- DELETE `/api/v1/biometric/record/:recordId` (501 not implemented)
- GET `/api/v1/biometric/health`
