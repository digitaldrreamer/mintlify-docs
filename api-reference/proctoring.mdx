title: "Proctoring"
description: "Sessions, events, streaming, and settings"
---

Use the OpenAPI playground to explore and try these endpoints with real responses. Most require `Authorization: Bearer <token>` and role checks.

- POST `/api/v1/proctoring/sessions` — start session
- POST `/api/v1/proctoring/sessions/:sessionId/end` — end session
- GET `/api/v1/proctoring/sessions/:sessionId/status` — session status
- POST `/api/v1/proctoring/sessions/:sessionId/events` — report event
- GET `/api/v1/proctoring/sessions/:sessionId/events` — list events
- GET `/api/v1/proctoring/events/stream` — streaming (placeholder)
- POST `/api/v1/proctoring/events/identity-verification`
- POST `/api/v1/proctoring/events/suspicious-behavior`
- POST `/api/v1/proctoring/events/application-switch`
- POST `/api/v1/proctoring/events/network-issue`
- GET `/api/v1/proctoring/settings/default`

See full request/response schemas in the OpenAPI spec.
